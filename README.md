# Лабораторная работа №2: Полиморфные последовательности (C++)

Демонстрация работы полиморфных контейнеров – динамического массива, связного списка и последовательности (Sequence) с поддержкой **изменяемых (mutable)** и **неизменяемых (immutable)** версий.  
Реализовано в рамках курса `"Информатика(полиморфные абстрактные типы данных)"` (2 семестр) на основе ТЗ.

## Структура проекта

- `DynamicArray.h` – шаблонный динамический массив с геометрическим ростом, операторами `[]`, `=`, энумератором.
- `LinkedList.h` – шаблонный односвязный список с аналогичным интерфейсом, добавлены методы удаления.
- `Sequence.h` – абстрактный интерфейс для линейных структур (чисто виртуальные методы).
- `ArraySequence.h` – реализация `Sequence` на основе `DynamicArray`; поддержка Mutable/Immutable через `instance()`.
- `ListSequence.h` – реализация `Sequence` на основе `LinkedList`; аналогичная поддержка Mutable/Immutable.
- `IEnumerator.h` – интерфейс для итераторов (энумераторов).
- `IndexOutOfRange.h` – класс исключения.
- `utils.cpp` / `utils.h` – свободные функции печати (`printSequence`), использующие энумератор и явное инстанцирование.
- `tests.cpp` / `tests.h` – модульные тесты (покрытие всех методов, стресс‑тест, таблица производительности).
- `menu.cpp` / `menu.h` – консольное меню для демонстрации работы (поддержка `int` и `std::string`).
- `main.cpp` – точка входа (запуск меню).
- `Makefile` – сборка проекта с флагами `-std=c++17 -Wall -Wextra -g -fsanitize=address`.

## Ключевые особенности реализации

### 1. Отсутствие ввода-вывода в заголовочных файлах АТД
- Все заголовки (`DynamicArray.h`, `LinkedList.h`, `Sequence.h`, `ArraySequence.h`, `ListSequence.h`) **не содержат `#include <iostream>`**.
- Функции печати (`printSequence`) вынесены в отдельный модуль `utils.h`/`utils.cpp` и используют **энумератор** для обхода элементов.  
  *Почему энумератор?* – он работает одинаково эффективно для массива и списка (O(n) против потенциального O(n²) при использовании `get(i)` в списке) и демонстрирует практическую пользу итераторов.

### 2. Разделение Mutable и Immutable через `instance()`
- Базовые классы `ArraySequence` и `ListSequence` содержат чисто виртуальный метод `instance()`.
- В `Mutable…Sequence` `instance()` возвращает `this` (изменяет текущий объект).
- В `Immutable…Sequence` `instance()` возвращает копию `*this` (создаёт новый неизменяемый объект).
- Все методы `append`, `prepend`, `insertAt`, `clear`, а также `removeFirst/Last/At` используют `instance()` – код пишется один раз, поведение зависит от наслединка.

### 3. Оптимизированные операции `concat`, `getSubsequence`, `map`, `where`
- **`concat`** создаёт новый изменяемый объект (`Mutable…Sequence`) и последовательно добавляет элементы через `append` – без лишних копирований и очисток.
- **`getSubsequence`** также возвращает `Mutable…Sequence`, заполняя её элементами из исходной последовательности.
- **`map`** – шаблонный метод, возвращающий новую последовательность типа `Mutable…Sequence<U>`, принимает функцию `(T, int) -> U` (доступен индекс).
- **`where`** – фильтрация, возвращает `Mutable…Sequence<T>`.
- Все эти методы **не изменяют исходный объект** (даже для Mutable).

### 4. Энумераторы
- Реализованы внутренние классы `DynamicArrayEnumerator` и `LinkedListEnumerator`.
- Используются в тестах и меню для вывода, а также в `printSequence`.
- `IEnumerator<T>` гарантирует полиморфное перечисление элементов.

### 5. Перегрузка операторов
- `operator[]` (чтение/запись), `operator=`.
- Все операторы вынесены из тел классов в отдельные реализации.

### 6. Исключения
- `IndexOutOfRange` выбрасывается при недопустимых индексах во всех классах.

### 7. Тесты (`tests.cpp`)
- **Общие тесты** для интерфейса `Sequence` (проверка всех методов, включая исключения).
- **Отдельные тесты** для `map`, `reduce`, `where` для `ArraySequence` и `ListSequence`.
- **Стресс-тест** – 100 тыс. операций `append` для каждого из четырёх типов (MutableArray, MutableList, ImmutableArray, ImmutableList) с замером времени и выводом сравнительной таблицы.
- **Отсутствие повторяющегося кода** – общие проверки вынесены в шаблонные функции.
- Все тесты проходят, **утечек памяти нет** (проверено AddressSanitizer и Valgrind).

### 8. Консольное меню
- Поддержка типов `int` и `std::string`.
- Выбор контейнера: `MutableArraySequence`, `MutableListSequence` (Immutable по желанию можно добавить).
- Все операции: `append`, `prepend`, `insertAt`, `removeFirst`, `removeLast`, `removeAt`, `set`, `get`, `clear`, `map`, `reduce`, `where`.
- После `map` и `where` показывается результат и запрашивается подтверждение замены текущей последовательности.
- Обработка ошибок ввода и исключений.

## Примечания

    - В заголовках АТД нет `#include <iostream>`.

    - Функции печати вынесены в отдельный модуль и используют энумератор.

    - Методы concat, getSubsequence, map, where оптимизированы и не содержат квадратичных сложностей.

    - Поддерживаются как изменяемые, так и неизменяемые последовательности, причём неизменяемость реализована через создание копий в каждом изменяющем методе.

    - Исходный код полностью соответствует принципам ООП, шаблонам C++17 и стандартам кодирования.
